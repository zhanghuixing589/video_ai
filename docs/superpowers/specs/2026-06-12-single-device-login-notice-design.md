# 单设备登录失效提示设计

## 目标

当当前浏览器的登录会话被另一设备的新登录替换后，下一次受保护请求返回 401，前端跳转到登录页，并可靠显示一次：

> 您的账号已在其他设备登录，请重新登录

## 根因

当前 Axios 响应拦截器先调用 Ant Design `message.error`，随后立即通过 `window.location.href` 刷新页面。页面卸载时消息容器被销毁，因此用户只能看到跳转，看不到提示。

## 方案

1. 新增一次性认证提示工具，使用 `sessionStorage` 保存提示代码，不保存完整服务端文本。
2. Axios 拦截器识别后端 401 中的“其他设备登录”，清除本地认证状态，写入 `SESSION_REPLACED` 提示代码，再跳转 `/login`。
3. 普通 401 只清除认证状态并跳转，不写入“其他设备登录”提示。
4. 登录页挂载后读取并删除一次性提示代码，通过当前 Ant Design App 消息实例显示中文提示。
5. 主动退出不写入任何错误提示。

## 数据流

`受保护请求` → `401 + 其他设备登录` → `清理 token/user` → `sessionStorage 写入提示代码` → `跳转登录页` → `读取并删除提示代码` → `显示一次错误提示`

## 测试

- 写入并消费 `SESSION_REPLACED` 后返回正确提示。
- 提示被消费后再次读取返回空，保证只显示一次。
- 未知提示代码不显示消息。
- 现有认证会话测试、全量 ESLint 和生产构建继续通过。
