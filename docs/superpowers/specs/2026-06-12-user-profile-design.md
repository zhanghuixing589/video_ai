# 用户个人中心设计

## 目标

为已登录用户提供统一的个人中心，首版完成：

- 查看当前账号基本资料。
- 修改显示名称和邮箱。
- 上传、替换本地头像。
- 校验旧密码后修改密码。
- 根据角色展示不同的功能标签。
- 为普通用户的浏览痕迹保留后续扩展框架。

观看历史、收藏和点赞的数据模型及业务接口不在本次实现范围内。

## 页面结构

个人中心使用统一顶部导航、用户摘要区和标签内容区。

用户摘要区展示头像、显示名称和角色。头像不存在时使用 Ant Design 默认头像。

角色对应的标签如下：

| 角色 | 标签 |
| --- | --- |
| `STUDIO` | 个人信息、我的视频、安全设置 |
| `USER` | 个人信息、浏览痕迹、安全设置 |
| `REVIEWER` | 个人信息、安全设置 |
| `ADMIN` | 个人信息、安全设置 |

“个人信息”展示用户名、显示名称、邮箱和角色，并提供编辑资料与修改密码入口。

“我的视频”复用现有 `GET /contents/mine` 接口，展示当前制片厂的作品名称、类型和状态。没有作品时显示空状态。

“浏览痕迹”首版只显示清晰的空状态和功能说明，不创建虚假记录。后续观看历史、收藏、点赞实现后再接入真实数据。

“安全设置”提供修改密码和退出登录。当前系统没有可靠的最后登录时间和设备记录，因此首版不展示伪造数据。

## API

### 获取资料

`GET /users/me/profile`

返回当前认证用户的 `UserDTO`，包含 `avatarUrl`。

### 修改资料

`PATCH /users/me/profile`

请求字段：

- `displayName`：必填，去除首尾空格后长度为 2 至 50。
- `email`：必填，规范化为小写并校验格式、长度和唯一性。

用户名和角色不可通过个人中心修改。邮箱被其他账号占用时返回 `409 Conflict`。

### 上传头像

`POST /users/me/avatar`

使用 `multipart/form-data`，字段名为 `file`。

- 支持 JPEG、PNG、WebP。
- 最大 5 MB。
- 使用服务端生成的随机文件名，不信任客户端文件名。
- 保存目录为可配置的 `uploads/avatars/`。
- 数据库存储可访问地址，例如 `/api/uploads/avatars/<filename>`。
- 替换头像成功后，仅删除头像目录内属于当前应用管理的旧文件。
- 校验失败或文件写入失败时不修改数据库。

Spring MVC 增加 `/uploads/**` 静态资源映射。头像目录和公开 URL 前缀通过应用配置集中管理。

### 修改密码

`PATCH /users/me/password`

请求字段：

- `oldPassword`
- `newPassword`
- `confirmPassword`

服务端校验旧密码、两次新密码一致、新密码长度 6 至 128，并拒绝新旧密码相同。修改成功后清除该用户当前服务端会话，前端清理当前标签页认证状态并跳转登录页。

## 数据与会话同步

`UserInfo` 和 `UserDTO` 均包含可选的 `avatarUrl`。

资料或头像修改成功后，前端以接口返回的最新用户对象更新页面状态和 `sessionStorage.user`，确保首页和个人中心使用一致资料。

登录响应和 `/auth/me` 也返回 `avatarUrl`，避免重新登录后头像丢失。

## 前端交互

- 新增受保护路由 `/profile`，所有已登录角色可访问。
- 首页及各工作台提供“个人中心”入口。
- 编辑资料使用弹窗表单，保存期间禁用重复提交。
- 头像上传前在浏览器校验类型和 5 MB 大小限制，上传后立即刷新头像。
- 修改密码使用独立弹窗，包含当前密码、新密码和确认密码。
- 接口错误通过现有 `getApiErrorMessage` 显示用户可理解的信息。
- 页面包含加载、空数据和提交中状态，并适配窄屏。

## 后端边界

- Controller 负责认证用户解析、multipart 接收和响应封装。
- UserService 负责资料唯一性、密码规则及用户更新。
- AvatarStorageService 负责文件类型验证、路径生成、文件保存和受控删除。
- Repository 继续承担用户和作品查询。

头像上传不接受外部 URL，不允许路径穿越，也不根据文件扩展名单独判断类型。

## 测试

后端测试覆盖：

- 当前用户资料读取。
- 显示名称和邮箱规范化更新。
- 邮箱冲突不更新资料。
- 旧密码错误、确认密码不一致、新旧密码相同。
- 密码成功更新并清除会话。
- 头像文件类型、大小和随机文件名校验。
- 替换头像时受控删除旧文件。

前端测试或可独立单元测试覆盖：

- 角色到标签列表的映射。
- 头像文件类型和大小校验。
- 更新后的用户资料写回 `sessionStorage`。

最终验证包括 Maven 测试、前端 Vitest、TypeScript 检查和 Vite 生产构建。
